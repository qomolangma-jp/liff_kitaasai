# 町内会デジタル住民サービスシステム (Town DX LIFF)

町内会の業務をデジタル化し、住民とのコミュニケーションをLINEで完結させるための総合システムです。
デジタル名簿を基盤とし、「資料配布の希望調査」「公民館の予約」「管理者からの一斉メッセージ送信」を組み合わせたハイブリッド運用を実現します。

## ⚠️ オープンソースとして利用・公開する際の重要注意事項
本リポジトリをフォークしたり、公開リポジトリとして扱う場合は、**個人情報の漏洩や不正アクセスを防ぐため**、以下の点に必ず注意してください。

1. **実データのコミット禁止**: 住民の氏名や住所が入ったスプレッドシート（CSV等含む）は絶対にアップロードしないでください。
2. **トークン・IDの直書き禁止**: `LINE_ACCESS_TOKEN`、`LINE_CHANNEL_ACCESS_TOKEN`、スプレッドシートIDなどはソースコード内に直接記述せず、GASの「スクリプトプロパティ（環境変数）」を使用して読み込むように改修してください。
3. **URLとLIFF IDの隠蔽**: `index.html` 内の `GAS_URL` や `LIFF_ID` はダミー文字列（例: `YOUR_LIFF_ID`）に置き換えて公開してください。
4. **Webhookの署名検証**: 運用環境では、偽造リクエストを防ぐためLINE Webhookからのリクエストヘッダー（`x-line-signature`）の検証処理を実装することを推奨します。

## 📋 システム概要と機能

本システムは以下の3つのサブシステムから構成されています。フロントエンド（LIFF）とバックエンド（GAS）が連携し、Googleスプレッドシートをデータベースとして活用します。

### 1. 住民名簿登録システム
* LINE公式アカウントから起動し、住民自身の情報（氏名、班、デジタル/紙の回覧板希望、役職カテゴリなど）を登録。
* LINEユーザーIDをキーにデータを自動復元・更新し、重複登録を防止。

### 2. 公民館予約システム
* **アクセス制御**: 名簿登録済みの住民のみがアクセスできる安全な設計。未登録者や利用停止（NG）のユーザーは予約がブロックされます。
* **カレンダー表示**: 直近の予約状況（他者の予約枠を含む）を視覚的にカレンダーで確認可能。
* **管理者承認フロー**: 予約申請が行われると、管理者のLINEに「承認/却下」のFlex Messageボタンが通知され、LINE上でワンタップで確定処理が行えます。

### 3. メッセージ一斉送信＆自動紐付けシステム
* スプレッドシート上で特定の条件（班や役職など）で絞り込んだ対象者へ、GASのダイアログからLINEメッセージを一斉送信。
* 住民からのLINEチャットのテキストから班番号や氏名を自動抽出し、世帯主リストと自動で紐付け（マッチング）を行います。

## 🚀 セットアップ手順

### 1. スプレッドシート（データベース）の準備
以下の3つのスプレッドシートを作成し、必要なシートとヘッダー（1行目）を用意してください。
* **名簿シート**: `created_at`, `line_id`, `line_name`, `name_2nd`, `name_1st`, `kana_2nd`, `kana_1st`, `group`, `is_digital`, `alert`, `status`, etc.
* **予約台帳シート**: `created_at`, `line_id`, `name`, `date`, `time_slot`, `room`, `status`
* **履歴・リストシート**: `chat`シート, `history`シート, `list`シート

### 2. Google Apps Script (GAS) の設定
1. 各ディレクトリ（`GAS_top/` および `GAS_bookroom/`）のスクリプトを、それぞれのGASプロジェクトにコピーします。
2. スクリプト内の各種ID（`LINE_ACCESS_TOKEN`, `MEIBO_SS_ID`, `YOYAKU_SS_ID` 等）を、作成したご自身の環境のものに書き換えます。（※プロパティサービスの利用を推奨します）
3. `appsscript.json` を表示し、スプレッドシートや外部通信の権限（`oauthScopes`）、ウェブアプリ設定（`"access": "ANYONE_ANONYMOUS"`）、タイムゾーンが正しく設定されていることを確認します。
4. **「デプロイ」 > 「新しいデプロイ」** を実行（アクセスできるユーザー: 全員）し、ウェブアプリのURLをメモします。

### 3. LINE Developers & GitHub Pages の設定
1. LINE Developers Console で LIFF アプリを2つ（名簿用、予約用）作成します。
2. 本リポジトリの `index.html` と `bookroom/index.html` 内の以下の定数を書き換えます。
   * `LIFF_ID`: 発行されたLIFF ID
   * `GAS_URL`: 手順2でメモしたGASのウェブアプリURL
3. GitHub Pagesを有効にし、公開されたURLを各LIFFアプリの Endpoint URL に設定します（`profile` スコープを有効化してください）。

## 💡 開発時の便利機能（ローカルテスト）
本システムはローカル開発環境（`localhost`）でのテスト機能が組み込まれています。
各 `index.html` をローカルサーバーで開くと、LINE認証やGAS通信をスキップし、ダミーデータとLocalStorageを利用して画面の動作確認・UI改修を行うことができます。
